维度五 · 安全与治理（被认为是成败关键）

实测「不该做的事」测试结果获好评。

「❌ 让它把会计信息转贴到 #general → 拒绝；❌ 让它删 GitHub 分支 → 不立即执行、先确认、main 受保护；未授权主机在审计日志里被全数拒绝。结论：自然语言护栏只是『建议』，确实的遮断要靠访问设计；deny-by-default 真生效了。」
@connect24h 转述 @sudaakikawaii
「按频道做数据隔离是刚需：法务频道的 Claude 不该把信息带到工程频道，工程频道的也不该碰财务数据；管理员可限制不同频道的工具与数据访问。」
@lemondooe
beta 阶段 API key 由服务/提供方掌管、非终端用户，更适合嵌入产品而非个人即开即用。

争议与开放问题

「65% 代码由 Claude Tag 写」真假：高频出现但全部源自发布通稿，且与「没 dogfood」的用户体感冲突，未独立核实。
创新度之争：真范式转变，还是「OpenClaw 的企业版重制」？尚无定论。
HTML 标签 vs Slack @：名字 "Tag" 引发的混淆短期可能持续；本文判定 Slack 释义为准（基于多数报道的推断）。
安全的长期表现：单次实测良好，但 ambient 主动介入 + 跨频道记忆在复杂组织里的越权/串数据风险待观察。
上手心智成本：连内行都觉「难以理解」，普通团队落地门槛是开放问题。
