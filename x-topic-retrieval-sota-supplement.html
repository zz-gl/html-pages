🕸️ 一、图检索（GraphRAG / 知识图谱）

⚖️ 核心权衡：「NaiveRAG 快但笨；GraphRAG 聪明但慢」——强在关系/全局，代价是构图成本、延迟、复杂度。🔗@akshay_pachaar
是什么：传统 RAG 把文档当孤立块；GraphRAG 抽成「实体-关系」图，按图结构检索/推理，答关系型与跨文档全局问题。🔗@pvergadia
✅ 适用：跨文档关联/全局总结/多跳推理、强可解释与降幻觉的合规场景（如 ISO/IEC 合规）。🔗@TheYotg
⚠️ 慎用：简单事实查、低延迟高频查——构图维护不划算，回到混合检索即可。
进展：Deep GraphRAG（分层检索+自适应重排，蚂蚁+浙大）、不依赖图数据库的 Graph RAG、多 Agent（向量+图+混合）、Weak-to-Strong GraphRAG(ICLR'26)、首篇综述。🔗@TheYotg @dair_ai @omarsar0
🚫 误区：GraphRAG 不是银弹——「一年多前是最热话题」，热度退去后回归理性，是补充非替代。构图依赖 LLM 抽取，抽错就检索错。🔗@TheYotg

🖼️ 二、多模态检索（深入）

🔄 范式转变：从「切块」到「看整页」——视觉 RAG（V-RAG/VisRAG）直接把整页当图像索引，用多模态模型同时理解文本+视觉，更接近人类看文档的方式。🔗@shao__meng
为何需要：传统切块处理图片/图表/表格吃力，还丢标题/字号/版面等视觉线索。🔗@shao__meng
关键资产：ColPali/ColQwen（后期交互+多模态，把 ColBERT 多向量用到页面图像）；VisRAG（多模态文档视觉 RAG）；MegaRAG（从视觉文档自动建知识图谱）。🔗@victorialslocum @kalyan_kpl @TheYotg
✅ 适用：扫描件/PDF/富版面/图表表格密集（财报、合同、手册、票据）——传统文本分块丢信息最严重处。代价：图像索引存储/算力更高。

🔇 三、无声失败（silent failure）怎么处理

⚠️ 最危险处：答案「听起来对」≠ 对，而失败不报错。「你的 RAG 可能在自信地犯错，而你毫不知情。」🔗@Umesh__digital · @cdiamond
建黄金集 + 测对指标：弃 BLEU，测 context recall / precision / faithfulness / coverage·diversity，而非「找第一名」。🔗@DataScienceDojo @HamelHusain
Ragas + 追踪平台（LangSmith 等）做离线+在线评测，量化每次检索/回答。🔗@CobusGreylingZA
LLM-as-judge：大模型当裁判批量打分「召回相关吗/答案有据吗」，把无声失败变可见。🔗@DataScienceDojo
强制引用/溯源：答案必须指回检索源；无源头 = 疑似幻觉。
在目标规模上评测：「5k 文档 90%，扩到 500k 就掉」——小数据成绩会骗你。🔗@akshay_pachaar
线上行为当代理信号：重问/转人工/点踩/追问 = 检索没召回到，接监控。
「不知道」兜底 + 置信阈值：召回不足宁可说没找到，别强答。
端到端 tracing：记录 查询→召回哪些块→用了哪些→答案，失败可回溯。
🧭 三块速记：图检索=关系/全局/多跳/合规上 GraphRAG（聪明但慢，补充非替代）；多模态=富版面/扫描件用 V-RAG/ColPali 把整页当图像别切块；无声失败=黄金集+context recall/faithfulness+Ragas/LLM-judge+强制引用+目标规模复测+「不知道」兜底+全链路 tracing，核心是让失败可见、可测、可回溯。
