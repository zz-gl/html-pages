自建 harness，还是
直接用 OpenClaw？

社区的真实答案不是「二选一」，而是 「在 OpenClaw 上学，最后搭你自己的」。核心权衡是一句话：用现成 = 省「管道工」时间、靠社区生态；自建 = 换「为我量身定制」+ 数据产权 + 模型自由，代价是维护成本会反噬。

🦞 直接用 OpenClaw（现成）

  • 省去自己维护底层「管道」的时间
  • 开箱即用、渠道预配好（iMessage/Discord）
  • 社区生态持续围着它造东西
  • 门槛低，把非开发者也带进了 agentic
  • 痛点：太复杂、升级即崩、记忆泄漏、安全存疑

🛠️ 自建 harness

  • 为「我」量身定制，而非取悦最多用户
  • 不依赖单一公司、可自由换模型
  • 数据留在自己手里、不被画像/训练
  • 长期 alpha 在「好到不开源」的私有方案
  • 痛点：维护量会爆炸、最终可能退居二线
决策法则（社区共识）：单一、你极熟、要长期打磨的工作流 → 自建（但建议站在 OpenClaw / SDK 肩上 fork，别从零写）；用例面很宽、想快速见效 → 用现成。怕锁定？锁定主要来自「围着某 harness 的独家功能建工作流」，而非模型本身——换模型常常只是一行 env var。
